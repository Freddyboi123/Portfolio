# Security
This week focused on security, specifically `hashing` and `token-based authentication`.

Initially, I underestimated the complexity of these topics. While implementing hashing and JSON Web Tokens (JWT), I encountered several issues, primarily due to my limited understanding of the provided security library. My first approach relied too heavily on copying existing code, which led to shallow understanding and recurring errors.

I realized that a functional implementation is not enough in security-related systems, where correctness and understanding are critical. As a result, I decided to restart my implementation from scratch.

By rewriting the code myself, I gained a deeper understanding of key concepts such as password hashing, secure storage, and token generation. In particular, I worked with how JWTs are structured (header, payload, signature), and how claims like username and roles are embedded and signed to ensure integrity.

The rewrite also gave me the opportunity to reduce my dependency on external libraries. By moving the functionality into my own project and rewriting the code, I was able to adapt the implementation to fit my specific entity structure. This allowed me to avoid being constrained by a predefined `UserDTO` structure and instead design a solution that better matched my application’s needs.

# Reflection
to quickly sumorize what this exercise has taught me
- **Reading Documentation** – Starting a project with a thorough review of the documentation can save a significant amount of time and prevent misunderstandings later in the development process.

- **Hashing** - Hashing is a relatively simple feature to implement, but it adds an important layer of security by ensuring that sensitive data, such as passwords, is not stored in plaintext.

- **Tokens** - Although more complex to implement than hashing, tokens provide a practical and powerful way to manage authentication and authorization. They add flexibility and control to how users interact with the system.

- **Delete Your Darlings** - Even if a lot of time has been invested in a solution, it is sometimes necessary to discard it and start over in order to achieve a better understanding and a more robust implementation.
